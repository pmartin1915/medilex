Fix: Remove eager errorLogger export causing runtime initialization error

PROBLEM:
- App crashed on startup with "Cannot read property 'S' of undefined"
- Error occurred during module loading, before React Native runtime was ready
- Grey screen on both iOS and Android

ROOT CAUSE:
- Line 137 in src/utils/errorLogger.ts created an instance at import time:
  `export const errorLogger = getErrorLogger();`
- This executed before AsyncStorage was initialized
- Caused "runtime not ready" error

SOLUTION:
- Removed eager export from errorLogger.ts (line 137)
- Now only exports getErrorLogger() function
- Instance created lazily when first accessed
- All consuming files already using getErrorLogger() - no changes needed

CHANGES:
- src/utils/errorLogger.ts: Removed line 137 (eager export)
- index.ts: Added extensive logging for debugging
